# Suggestions Register

This is the canonical backlog for new ideas, enhancements, and future capabilities for the Agent Mia platform. Planning documents should reference this file.

## Near-Term Suggestions
- Add a dedicated Developer Agent with clear handoff rules to Gunther and the Environment & Release Manager
- Build a Gitea Task Board skill for issue creation, assignment, board movement, and stage labels
- Build a Gitea QA Evidence skill so Gunther can require linked evidence before approval
- Build a Gitea Release Flow skill to validate promotion rules and release readiness
- Add issue and pull request templates in the repository for feature, bug, sprint task, release, and hotfix work
- Add a standard Definition of Ready and Definition of Done for every sprint task

## Platform Suggestions
- Add a lightweight dashboard for sprint status, issue ownership, blocked work, and release readiness
- Add artifact storage for test evidence, screenshots, logs, and release notes
- Add central logging for OpenClaw activity and agent decision logs
- Add metrics and monitoring for Ollama, OpenClaw, and the Lume VM
- Add backup and restore procedures for the Lume VM and agent configuration files

## Identity and Operations Suggestions
- Create a managed Apple Account for Mia only if macOS-specific services are actually needed inside the VM
- Create a managed Google account for Mia for Gmail, Drive, Calendar, and Google Voice workflows if those services become part of her operating model
- Use a dedicated virtual phone number only if required for account recovery, notifications, or service integrations
- Keep all Mia accounts documented as managed service accounts under Thomas family control

## Advanced Suggestions
- Add automated sprint review summaries generated by Mia
- Add automated defect triage and severity proposals generated by Gunther
- Add release checklists that block promotion until all evidence is attached
- Add change logs and decision logs that sync to Gitea issues or repository docs
- Add a node or secondary environment for browser automation if the VM later needs to stay isolated from the main Gateway
